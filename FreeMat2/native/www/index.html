Introduction

FreeMat is a free environment for rapid engineering and scientific prototyping and data processing. It is similar to commercial systems such as MATLAB from Mathworks, and IDL from Research Systems, but is Open Source. FreeMat includes several novel features such as a codeless interface to external C/C++/FORTRAN code, parallel/distributed algorithm development (via MPI), and plotting and visualization capabilities. FreeMat is available under the GPL license. Supported platforms include Linux, Windows and Mac OS X.

FreeMat 2.0 Release Candidate 3-12-2006 

Its been nearly a year (!) since the last release of FreeMat. A new version is now ready for testing. I have elected to release it as a Release Candidate instead of a full release because of the significant changes that have been made since version 1.10. Here is just a short list of the new features.

  • Handle-based Graphics including 3D surface plots, multiple plots per figure, and full customization of axes properties. Preliminary support for OpenGL rendering.
  • Support for MATLAB-style classes and object oriented programming, including operator overloading.
  • New GUI interface based on Qt4. FreeMat now sports a new terminal interface, a highlighting editor, and some new tools.
  • Speed improvements and optimizations (some loops are 10-100 times faster).
  • Preliminary support for reading and writing MATLAB .MAT files.
  • PDF output on all platforms.
  • New build system based on qmake.
  • Many new MATLAB-compatibility functions and scripts.
  • Native Aqua application on Mac OS X.
  • Linux distribution now includes a binary release that should run on several distributions.
  • Over 200 new pages of documentation, and an improved built in help browser.
The new release of FreeMat is under the GNU Public License. The following features are no longer supported or are not supported in this Release Candidate.
  • The copy function has changed for figures. It is now based on bitmaps, even on Win32 (where it used to copy a vector drawing).
  • The MPI interface does not build in this RC1. I will fix this for the 2.0 release.
Thanks to everyone who contributed to this release (see the About FreeMat dialog for a list).

New release of FreeMat 3-14-2005 

A new release of FreeMat is available for download (version 1.10). Please consider making a donation to the project - it would help offset the costs of developing and maintaining FreeMat for multiple platforms (particularly the non-Linux ones). This version features significant improvements over version 1.09. The major changes are listed below.

  • Sparse matrix support - Thanks to Al Danial for help with this one. FreeMat now supports most of the sparse matrix functionality of Matlab, including the creation and manipulation of sparse matrices, the solution of sparse linear problems (via UMFPACK), and sparse eigenproblems (via ARPACK).
  • More linear algebra - Including LU factorizations for full and sparse (square) matrices.
  • Debug support - You can now debug FreeMat routines efficiently using the dbstep, dbstop, dbauto, dblist and other debug related routines. See the chapter on Debug functions in the manual for more information. A debug tutorial may appear in a future version.
  • Improved compatibility - A number of support functions (e.g., mod, unique, sort, etc.) are now available in FreeMat
  • Boolean shortcut evaluation - FreeMat now implements boolean shortcut expressions
  • GUI Enhancements - The GUI has some usability enhancements (partial command recall, font size preference, etc).
  • Many, many bug fixes - Thanks to Jeff Fessler, Bruno De Man, Brian Yanoff and others for the bug reports and scripts.

Compatibility Warning! 10-12-2004 

Just to reemphasize a couple of issues that have come up since the 1.08 release.

  • There was a bug in 1.07's save command that makes the files not load correctly in 1.08 (and not even in 1.07 under certan circumstances). I do not have a conversion utility available, but if there is demand for one, I can provide one.
  • The documentation for the image command stated that the default zoom for an image was 0, when in fact it was 1. In 1.08, the default zoom is 0, as was claimed in the documentation. This may cause some problems if you wrote routines that assumed it was 1.
  • The functions newplot,useplot,newimage,useimage,printplot,printimage have all been replaced with figure and print. While no backwards compatibility scripts were provided (and indeed, the exact behavior of the original functions is difficult to reproduce), I can make such scripts available if required.
  • Do not close the GUI window to quit FreeMat. Use the quit command. This bug will be fixed in the next release.

New release of FreeMat 10-10-2004 

A new release of FreeMat is available for download (version 1.08). There are significant changes under the hood, but from a user's perspective, this is primarily a bug fix and feature release. There are a few changes that are noticeable, however, and they are listed here.

  • The functions newplot,useplot,newimage,and useimage have all been replaced with the figure command (similar to Matlab).
  • A new cross-platform GUI is now available on all supported platforms. For Linux/Unix variants, try starting FreeMat with a "-gui" option to access the new GUI.
  • Online help is now available on OS X.
  • The X11 server is no longer used on OS X.
  • Figure windows on Win32 no longer have "save" and "copy" options. This functionality is achieved using the "print" and "copy" commands, respectively.
  • FreeMat now uses the excellent FLTK (http://www.fltk.org) toolkit to provide graphical services.
  • A number of bug fixes and feature enhancements.
Special thanks to Bruno DeMan for extensive testing, suggestions, bug reports, and an improved README!

New release of FreeMat 08-26-2004 

A new release of FreeMat is available for download (version 1.07). This version features a number of significant enhancements over 1.06. Among the improvements:

  • Support for singular value decompositions (SVD) both compact and full size.
  • QR decompositions (compact and full size) for all matrix types.
  • Eigendecompositions and generalized eigendecompositions for symmetric and non-symmetric matrices.
  • Enhanced plot functions, including a legend command.
  • Special functions added, such as the gamma function, error function, dawson's function, among others.
  • A set of new random number generators for chi-square, Poisson, multinomial, and other distributions.
  • A p-code mechanism to byte-compile FreeMat scripts to binary form (only works with Linux currently).
  • Build stand-alone binaries out of FreeMat scripts (only on Linux currently).
  • Routines to convert numerical arrays into binary and reverse.
  • Better error reporting.
  • A number of additional bug fixes.

Initial release of FreeMat 03-31-2004 

The first version of FreeMat has been released! FreeMat is an interactive matrix-driven environment for engineering and scientific data analysis (similar to programs like MATLAB, IDL and Octave). FreeMat features a high degree of MATLAB compatibility, along with features from IDL such as pass-by-reference, along with unique features such as codeless interfaces to C, C++ and FORTRAN. FreeMat includes online help and a large array of useful built in functions. An installer is available for Windows. Source code is available for Unix/Linux/MacOSX. An installer for Mac OS X will be released soon.
